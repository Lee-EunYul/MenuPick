# MenuPick

오늘의 식사 결정을 빠르게 도와주는 **메뉴 추천 웹앱**입니다.  
질문 기반 추천, 위치 기반 룰렛, 선택 이력 분석을 통해 사용자가 메뉴를 더 쉽게 고를 수 있도록 만든 프로젝트입니다.

## 웹페이지

- 배포 주소(확인됨): https://menupick-web-06280859348.azurewebsites.net


## 1) 프로젝트 한 줄 소개
- **무엇을 해결하나요?**: "오늘 뭐 먹지?" 결정을 빠르게 내리기 어려운 문제
- **어떤 방식으로 해결하나요?**: 질문 응답 + 랜덤 룰렛 + 이력 기반 개인화 추천
- **프로젝트 성격**: Monorepo 구조의 Web + API 애플리케이션

## 2) 어떤 앱인가요?
- 웹에서 바로 사용하는 메뉴 추천 서비스
- 사용자 설정과 선택 이력을 반영해 추천 품질을 높이는 구조
- 로컬 개발, Docker 기반 로컬 인프라, Azure 배포(Bicep)까지 고려된 구성

## 3) 핵심 기능

### 3-1. 질문 기반 맞춤 추천
- 예산, 매운맛, 국물 유무, 식사 인원, 카테고리(총 5개 질문) 입력
- 조건 필터링 + 점수 계산으로 상위 메뉴 추천
- 추천 결과 선택/패스 기록 저장

### 3-2. 위치 기반 랜덤 룰렛
- 카테고리/반경을 설정하고 룰렛 실행
- 브라우저 위치 권한을 사용해 주변 후보를 반영
- API 실패 시 로컬 fallback 룰렛으로 동작 유지

### 3-3. 선택 이력 분석 추천
- 사용자 ID + 기간 기준으로 선택 통계 조회
- 수락률, 카테고리 분포, 최다 카테고리 분석
- 최근 선택 패턴을 바탕으로 추천 메뉴 제시

### 3-4. 개인화 추천(고도화 API)
- 사용자 설정(매운맛 상한, 선호 카테고리) + 최근 패턴 반영
- 거절 메뉴/최근 선택 메뉴를 제외하고 추천 결과 생성

### 3-5. 설정 화면
- 사용자 ID, 선호 카테고리, 매운맛 레벨, 알림 시간 관리
- 위치 권한 상태 확인
- 로컬 저장소 기반 설정 저장

## 4) 기술 스택

### Frontend
- Next.js 15
- React 18
- TypeScript

### Backend
- NestJS 10
- TypeScript
- Node.js

### Data / Storage
- 추천 메뉴 데이터: 코드 내 샘플 데이터(`SAMPLE_MENUS`) 사용
- 사용자 선택 이력: API 서버 메모리 배열 저장(`SelectionsService`), 현재 DB 미연동
- PostgreSQL 마이그레이션 SQL 파일 보유(`apps/api/src/database/migrations`)

## 5) 프로젝트 구조

```text
MenuPick/
├─ apps/
│  ├─ web/        # Next.js 프론트엔드
│  ├─ api/        # NestJS 백엔드
│  └─ api-func/   # Azure Functions 버전 API(별도)
├─ infra/
│  ├─ docker/     # 로컬 인프라(docker-compose)
│  └─ azure/      # Azure 배포(Bicep/스크립트)
└─ .github/       # 워크플로우
```
